How It Works: Three-Tier Architecture

Tier 1: Personal Ledger

Each citizen's phone stores encrypted IQD balance and transaction history. Peer-to-peer payments work offline via NFC/BLE.

Tier 2: Super-Peer Network

Federated super-peers in Baghdad, Basra, Erbil coordinate conflict resolution. Byzantine consensus (3-of-5) detects double-spends. No transaction fees.

Tier 3: CBI Control Layer

CBI controls monetary policy, issuance rates, KYC tiers. Real-time monitoring of money supply and velocity. Full AML/CFT visibility.

Offline-First: Works Without Connectivity

Most critical innovation for rural Iraq

NFC Peer-to-Peer

  • Two phones touch
  • Payer initiates payment
  • Receiver confirms receipt
  • Both create signed transaction
  • Works completely offline

Later Sync

  • When connectivity returns
  • Phone syncs to nearest super-peer
  • Super-peer verifies no double-spend
  • Transactions confirmed in ledger
  • Process takes <5 seconds

A rural farmer in Anbar can transact during market day with no cell service. When they reach town and get connectivity, their transaction is confirmed. Zero friction.

Risk Management & Safeguards

Network Resilience
Devices sync offline via NFC/BLE. If super-peers are down, transactions still work locally and confirm later.
Double-Spend Detection
Byzantine consensus (3-of-5 super-peers vote on transaction order). Automatic quarantine of conflicts for CBI review.
Key Recovery
Users designate 3-7 trusted contacts as recovery delegates. Lost phone? Recover account in 5 days via delegate approval.
Capital Controls
CBI maintains geofencing and transaction limits. Prevents unauthorized capital flight. Can be adjusted in real-time.
Regulatory Compliance
All transactions logged for AML/CFT. Real-time flag system for suspicious activity. CBI retains full control.

Legislation authorizing CBI to issue digital IQD. Defines legal status, consumer protections, tax treatment.

Public governance framework. Transparency on monetary policy, issuance schedule, security practices.

AML/CFT standards met (FinAction reference). FATF-compatible transaction reporting.

Risk Mitigation: Critical Issues Addressed

Cybersecurity Risk:
Decentralized super-peers (no single point of failure). Regular security audits. Hardware security modules for key storage.
Adoption Risk:
Phased rollout starting with government employees (built-in user base). Physical IQD remains in circulation for 2-3 years.
Network Outage Risk:
Offline-first design (NFC/BLE). Transactions work without super-peers. Delayed sync is acceptable in war/disaster scenario.
Political Risk:
Open source code (no vendor lock-in). CBI can switch operators if needed. Parliament oversight via law, not executive decree.
Inflation Risk:
CBI controls issuance schedule. System prevents unauthorized money creation. Real-time monitoring enables rapid response to inflation signals.
